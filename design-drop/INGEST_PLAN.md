# Prism web-shell ingest plan — drop-website redesign

Source: `design-drop/Drop Website.dc.html` (claude.ai design, 49 screens, single-file
interactive spec; screens rendered by `go('<name>')` router). Reference only — port
markup/CSS per screen, do NOT copy the SPA runtime.

## Scope (founder: "ingest everything" → everything that belongs to the static site)
IN (public + auth shells + acquisition, replaces existing pages):
home→index.html, discover(public)→events.html, event→event.html, artists→artists.html,
artist→artist.html, venues→venues.html, venue→venue.html, about, getapp→download.html,
promoters, legal→privacy.html+terms.html, link, login/signup/forgot/reset/verify/referral→account.html,
NEW: city.html + genre.html (SEO templates), share-plan.html + share-recap.html +
share-wrapped.html (share landings), 404.html, search (embedded in events.html).

OUT (signed-in app surfaces → app.trydropapp.com / drop-mobile-app web export, future work):
discover signed-in state, crew/plans, myshows/memories/recap/wrapped/seen/tagged,
wallet/paywall, profile/settings/editprofile/blocked/delete, wizard, notifications,
festival my-schedule, suggest, promoter dashboard/manage, admin. Their design rides the
same tokens; implement in the app repo later.

## Token map
Identity — the design was generated FROM the synced Prism design system; site.css
already carries the same :root block (generated by prism-tokens/sync.mjs). Component
classes (.wsh web-shell, .wsc show-card, .chip, .skel, .metric, .seg, .locchip, .dual
range slider, .empty) port from the dc.html <style> + drop-design components/app.css
into a new `shell.css` (site.css stays generated-only). No new raw hexes allowed.

## Rules
- Replace, don't add: each new page replaces the old file; delete retired css (keep
  account.css/link.css only if still referenced).
- Preserve function: keep data.js wiring, affiliate/ticket links, SEO meta + structured
  data, _headers/_redirects, favicon, analytics hooks from the CURRENT pages.
- Mobile: port the design's @media (max-width:720px) rules; thumb-zone sticky bar on event.
- Every page passes the 19 rules (foundations/design-rules.md in drop-design).
- Tests: Playwright smoke (tests/smoke.spec.ts) must pass; extend for new pages.
- Deploy: manual dist/ mirror + wrangler — ONLY after Arya's explicit go.

## Tracks (Foundation first, then parallel; one branch `redesign/prism-web-shell`, one PR)
0. Foundation: shell.css (web-shell component classes + shared layout + media queries),
   shared nav/footer markup pattern (per-page copy, matching design).
1. Core browse: index, events(+search), event, artists, artist, venues, venue.
2. Auth shells: account.html (login/signup/forgot/reset/verify/referral states).
3. Acquisition: city, genre, share-plan, share-recap, share-wrapped, download, about, promoters.
4. Static: privacy, terms, link, 404 + smoke-test extension + dist/ mirror refresh.
